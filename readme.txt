* Внешний вид

Диалог программы состоит из двух частей: списка жанров, и списка радиостанций данного жанра.
Будет разумно скопировать уже существующий интерфейс с популярной программы iTunes, за некоторыми доработками.
Перечень этих доработок будет сводится к более нагрядному хранению любимых радиостанций\жанров и работе со списком радиостанций
при перезагрузки приложения.
Кроме того, программа не должна без надобности лазять на сервер shoutcast за списком композиций. Поэтому прдпочтительно
использование кешированных данных.

- вариант использования:

При первом запуске загружается список жанров. При выборе определнного жанра происходит подгрузка списка радиостанций. Все данные
попадают в базу данных RhythmDB. Обновление данных происходит через нажатие на кнопку обновления путем удаления всех данных и
повторного их востановления по данным полученным с серверов shoutcast. Удаление не будет затрагивать любимые радиостанции.
Обновление будет происходить в момент старта, периодически раз в неделю если приложение не перезапускалось (все это
в ручную настраиваемые параметры, с возможностью полной блокировки обращений к shoutcast серверам).

При нажатии кнопки обновить, внешний вид обновления должно быть наиболее дружелюбным по отношению к пользователю. Удаление
приводит к очистке всех композиций с запоминанием активных позиций (выборов жанра, радиостанции точек просмотра). После ообновления
списков программа пытается восстановить указанные позиции.

Кнопки обновления должны быть доступны на обоих окнах. Верхнем - списке жанров. Можно разместить кнопку справа от текста общего числа
жанров.

All 6 genres                      (R)

И второго окна:

Dance (567)                       (R)
(o)  Some station name     bitrate      url
(*)   Some 2 some som      bitrate      url

, где (R) - кнопка обновления списков. (*), (o) любимая и не любимая радио станция.

Любимы и не любимый варинат влияет на сортировку (идут первыми), а так же на механизм обновления (их не затрагивает). 

После нажатия на кнопку обновления - создается независимый процесс (тред или колбэк) и начинает загрузку. В случае
ошибки сообщение преедается в управляющий процесс (интерфейс пользователя) и инициируется показ сообщения об ошибке.
В случае успеха значения точек просмотра и активные позиции запомнинаются. И создается независимый процесс. Когда процес
выполнен успешно. Список очищается, все кроме любимых станций. Чистится так же база данных. Изменения отражаются на интерфейсе
пользователя. Затем данные попадают в базу и на интерфейс пользователя. Программа автоматически старается востановить старые
позиции просмотра (точки просмотра и активные позиции (выделения))

- вариант использования:

При первом запуске загружается список жанров. При выборе определнного жанра происходит подгрузка списка радиостанций.
В кеше сохраняются радиостанции помечанные как любимые. Эти радиостанции имеют специальную отметку на списке радиостанций.
Подгрузка списков происходит после перезапуска приложения.
